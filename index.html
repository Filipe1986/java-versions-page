Java Versions & JEPs — Quick Guide

Overview of Java's release model, LTS vs non-LTS, and how JEPs shape new features and deprecations.

What changed in Java's release cadence?

Since Java 9, the JDK uses a time-driven release cadence: a new feature release every 6 months (March/September), with Long-Term Support (LTS) releases approximately every 3 years. This makes features arrive faster but encourages using LTS builds in production for stability.

Versioning model

  • Before Java 9: unpredictable major releases (e.g., Java 7, 8)
  • Since Java 9: semantic-like versioning but time-driven (e.g., 9, 10, 11, 12, ...). LTS: 8, 11, 17, 21, ...
  • Feature releases are supported for a shorter lifecycle; vendors provide LTS builds for production.

Key concepts

  • LTS (Long-Term Support) — chosen releases with extended vendor support.
  • Non-LTS — faster feature delivery, shorter support window.
  • JEPs — Java Enhancement Proposals: design documents that propose features, APIs, and changes.

How JEPs work (short)

JEP = proposal document that explains motivation, specification, and compatibility implications. A JEP can be:

  • Targeted for a specific JDK release
  • Marked as proposed, accepted, implemented, or targeted
  • Include early-access builds for feedback

Notable JEPs (examples)

  • JEP 223 — New Date/Time API (was JSR 310) (note: different historical numbering)
  • JEP 261 — Module system (part of Java 9)
  • JEP 355 — Text Blocks (preview -> standard)
  • JEP 409 — Sealed Classes (preview -> standard in later JDKs)
  • JEP 405 — Record Patterns & Pattern Matching improvements

Choosing a Java version

  • Use LTS (11, 17, 21...) for production unless you need a specific new feature.
  • For new projects, consider the latest LTS and test on newer non-LTS to adopt features earlier.

How to check your Java/JDK version

Run in a terminal:

java --version
javac --version

Example output (openjdk):

openjdk 17.0.8 2024-07-16
OpenJDK Runtime Environment (build 17.0.8+7-LTS)

Identifying LTS vs feature

  • Major version numbers in the table above — cross-check with vendor release notes.
  • Oracle and other vendors mark LTS in their downloads pages.

Small checklist for upgrading

  • Run tests on the target JDK
  • Check deprecated APIs and removed features in relevant JDK release notes
  • Verify third-party library compatibility (especially native code)
